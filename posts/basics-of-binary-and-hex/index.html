The Basics of Binary and Hex

Posted on Jul 21, 2020

Binary and Hexidecimal number systems are prevalent when discussing low-level computer programming and architecture. Let’s briefly discuss what they are, how they work, and why they’re used.

Decimal

Before we get to binary and hexidecimal numbers, let’s review our standard number system: decimal, or base-10.

In a decimal number we can use 10 different numerals: 0 through 9. Each “place” in a decimal number allows us to count in increments 10 times larger than the previous place.

For example: the number 25 counts 5 “ones” in the rightmost numeral, and 2 “tens” in the leftmost numeral, adding up to 25. The “places” look like this:

digit25
place101

Expanding further to the left, we have the “hundreds” place, the “thousands” place, and so on. Each place 10 times the previous one.

Binary

Binary numbers follow a similar rule, but each place is only 2 times larger than the previous one. Additionally, a binary (base-2) number is expressed with only two numerals: 0 and 1.

Counting the same number (25) in binary looks like this:

digit11001
place168421

You’ll see that to make 25 we have 1 sixteen, 1 eight, and 1 one.

We typically write binary numbers with a “0b” prefix. So 25 in binary would be “0b11001”. This makes it easier for us to know we’re talking about “twenty-five in binary” and not “eleven-thousand and one in decimal”.

Bitwise Operations

Let’s sidetrack for a moment to talk about the typical calculations that we perform with numbers. With decimal numbers we’re used to adding, subtracting, multiplying, dividing, etc. We can do all of those things with binary numbers as well, but binary numbers also allow us to easily perform a set of logical operations called “bitwise operations”. This is a shortcut for low-level boolean logic in computers.

AND (&)

Rules:

  • 0 & 1 => 0
  • 0 & 0 => 0
  • 1 & 1 => 1

Example: 0b01101 & 0b11011 == 0b01001

OR (|)

Rules:

  • 0 | 1 => 1
  • 0 | 0 => 0
  • 1 | 1 => 1

Example: 0b01101 | 0b11011 == 0b11111

XOR (^)

Rules:

  • 0 ^ 1 => 1
  • 0 ^ 0 => 0
  • 1 ^ 1 => 0

Example: 0b01101 ^ 0b11011 == 0b10110

Shift Left («)

Rule: Shift bits in left value to the left by the number of places specified by right value. “Empty” spots are filled with zero bits.

Example: 0b01101 << 0b10 == 0b110100

Shift Right (»)

Rule: Shift bits in left value to the right by the number of places specified by right value. “Empty” spots are filled with zero bits.

Example: 0b01101 >> 0b10 == 0b11

💡 It might help to pop open the dev tools JS console in your browser and play around with the preceding operators for a while until you're able to develop an intuition for how they work.

Hexidecimal

If decimal number places increment by 10, and binary number places increment by 2, it follows that hexidecimal number places should increment by 16. If each place allows us to count to 16, we’ll need to expand our numerals beyond just 0 through 9. To help us count to 16, we use the following letters to represent numbers:

letternumber
A10
B11
C12
D13
E14
F15

This time let’s look at 28 as an example:

digit1C
place161

To make 28, we have 1 sixteen and C (12) ones.

Hex numbers are typically written with a “0x” prefix, so 28 in decimal is 0x1C in hex!

Bits and Bytes

There’s one more thing we should talk about before wrapping this up. A bit is a single “place” in a binary value - it can be either 0 or 1. A binary number with 4 places, like 0b1111, is a 4-bit number.

A byte is (usually) 8 bits. So a number like 0b11110000 would be an 8-bit or 1-byte number. There are some historical exceptions to the size of a byte, so to be unambiguous we could call them “octets” as well.
